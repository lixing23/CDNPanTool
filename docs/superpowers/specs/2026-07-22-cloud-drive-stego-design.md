# 伪装图片云盘桌面工具设计

## 背景

当前项目是一个 Python Tkinter 桌面图床上传工具，核心文件是 `image_uploader_gui.py`。现有能力包括：选择图片、通过消费宝 OSS 签名接口上传、返回原始直链和水印链接、生成 Markdown/HTML/BBCode、保存本地上传历史。

本次改造目标是把它从单纯图床工具扩展为类似云盘的桌面分享工具：用户选择任意文件，程序把文件拼接到一张自定义封面图片后面，上传生成的伪装图片，并返回一段短分享文本。接收方使用同一个程序粘贴短分享文本，即可解析、下载伪装图片并提取原文件。

## 目标

第一版实现以下能力：

- 保留桌面工具形态，不引入 Web 服务。
- 支持上传任意类型文件。
- 支持设置默认封面图片。
- 支持单次上传时临时选择封面图片。
- 将原文件二进制拼接到封面图片后，生成伪装图片文件。
- 继续使用当前 OSS 签名上传流程上传伪装图片。
- 生成无需密码的短分享文本。
- 支持粘贴短分享文本解析直链。
- 支持下载伪装图片并提取原文件。
- 保存上传历史、分享文本、原文件名、文件大小、上传时间。

## 非目标

第一版不实现以下能力：

- 用户登录或账号体系。
- 在线文件管理后台。
- 真正短链服务。
- 分享过期时间。
- 访问密码。
- 云端删除文件。
- 多人权限管理。
- 文件夹同步。
- 增量上传或断点续传。

## 推荐方案

采用纯桌面工具方案。分享文本中包含经过压缩和 URL-safe Base64 编码后的直链与必要元数据。接收方无需访问额外服务器，只需要同一个桌面工具就能解析分享文本并下载文件。

该方案最符合当前项目基础，复用现有 OSS 上传能力，避免引入后端服务，也便于第一版快速落地。

## 总体流程

### 上传分享流程

1. 用户选择一个任意类型文件。
2. 程序选择封面图片：优先使用本次临时封面；若未选择，则使用默认封面。
3. 程序读取原文件信息，包括文件名、文件大小、MIME 类型和 SHA-256 校验值。
4. 程序生成伪装图片容器文件。
5. 程序把伪装图片上传到 OSS。
6. 程序生成短分享文本。
7. 用户复制短分享文本发给别人。
8. 上传历史写入本地 JSON 文件。

### 接收提取流程

1. 接收方打开桌面工具。
2. 粘贴短分享文本。
3. 程序解码分享文本，得到伪装图片直链和文件基础信息。
4. 程序下载伪装图片。
5. 程序从文件尾部读取容器索引。
6. 程序提取原文件二进制。
7. 程序校验 SHA-256。
8. 用户选择保存目录，程序写出原文件。

## 伪装图片容器格式

生成的伪装图片由四段组成：

```text
[cover image bytes]
[metadata json bytes]
[file bytes]
[trailer bytes]
```

### metadata

metadata 使用 UTF-8 JSON，字段如下：

```json
{
  "v": 1,
  "name": "example.zip",
  "size": 123456,
  "mime": "application/zip",
  "sha256": "...",
  "created_at": "2026-07-22T12:00:00+08:00"
}
```

### trailer

trailer 放在伪装图片末尾，用于快速定位 metadata 和 file bytes。

固定结构：

```text
XFBSTEG1|metadata_offset|metadata_size|file_offset|file_size|trailer_size
```

解析时从文件末尾读取一段固定窗口，查找 `XFBSTEG1|` 标记，再解析偏移量和长度。这样不需要扫描整个文件，也不会依赖封面图片格式内部结构。

## 分享文本格式

分享文本格式：

```text
xfb1.<payload>
```

payload 生成步骤：

1. 构造紧凑 JSON。
2. 使用 zlib 压缩。
3. 使用 URL-safe Base64 编码。
4. 去掉末尾 padding。

紧凑 JSON 字段：

```json
{
  "v": 1,
  "u": "https://example.oss/path/file.png",
  "n": "example.zip",
  "s": 123456
}
```

字段说明：

- `v`：分享格式版本。
- `u`：伪装图片直链。
- `n`：原文件名，用于解析前展示。
- `s`：原文件大小，用于解析前展示。

无需密码模式下，分享文本是混淆编码，不是强加密。它能让肉眼看不出原始链接，但任何拥有程序的人都可以还原直链。这符合第一版“无需密码、尽量短、看不出是链接”的目标。

## 模块划分

当前项目只有一个主文件，建议改造时拆成以下模块：

```text
image_uploader_gui.py
oss_client.py
stego_container.py
share_codec.py
settings_store.py
```

### image_uploader_gui.py

负责 Tkinter 界面、事件绑定、线程调度和状态展示。

界面改为两个主区域：

- 上传分享。
- 解析提取。

### oss_client.py

负责 OSS 相关能力：

- 获取上传签名。
- 上传文件。
- 拼接直链。
- 下载远程文件。

现有 `fetch_sign`、`http_post_multipart`、`join_url` 等函数迁移到该模块。

### stego_container.py

负责伪装图片容器：

- 生成伪装图片。
- 读取 trailer。
- 解析 metadata。
- 提取原文件。
- 校验 SHA-256。

### share_codec.py

负责分享文本：

- 编码分享文本。
- 解码分享文本。
- 校验前缀和版本。
- 处理 Base64 padding。

### settings_store.py

负责本地配置与历史记录：

- 默认封面路径。
- 上传历史。
- 解析历史。
- JSON 文件读写容错。

## 本地文件

建议本地生成以下运行时文件，继续在 `.gitignore` 中忽略：

```text
upload_history.json
settings.json
cache/
```

`cache/` 用于保存临时伪装图片和下载中的远程文件。成功上传或提取后，可以自动清理临时文件。

## 错误处理

需要明确处理以下错误：

- 未设置默认封面且本次未选择封面。
- 封面图片不存在或无法读取。
- 待上传文件不存在或无法读取。
- 上传签名接口失败。
- OSS 上传失败。
- 分享文本格式错误。
- 分享文本版本不支持。
- 下载直链失败。
- 下载到的文件不是有效伪装图片。
- trailer 缺失或损坏。
- SHA-256 校验失败。
- 保存目录不可写。

所有错误在 GUI 中展示友好提示，同时写入日志区域。

## 测试策略

第一版使用 Python 标准库测试即可，不新增第三方依赖。

重点测试：

- 分享文本编码和解码往返一致。
- Base64 padding 被正确恢复。
- 任意二进制文件拼接后可完整提取。
- 原文件名、大小、MIME、SHA-256 元数据正确。
- trailer 损坏时能报错。
- 分享文本前缀错误时能报错。
- 无默认封面时上传按钮给出明确提示。

## 兼容性说明

该方案依赖上传后的伪装图片保持原始二进制不变。如果 OSS 或中间平台对图片进行压缩、裁剪、重编码，拼接在尾部的文件数据会丢失。因此第一版中分享文本应指向 OSS 原始文件直链，不应使用会触发图片处理的水印链接或缩略图链接。
